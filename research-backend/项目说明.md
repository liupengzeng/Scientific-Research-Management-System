项目名称：高校科研管理系统
一、 项目概述
本项目旨在开发一个现代化、高效率、高可用的高校科研信息管理平台。系统将服务于科研管理人员、教师（科研人员）、学院领导等不同角色，实现对科研项目、科研成果、科研经费、学术活动等核心业务的数字化、流程化、规范化管理。

三、 系统架构设计
采用经典的前后端分离架构。

前端 (Frontend)： Vue 3单页面应用，负责页面渲染和用户交互，通过Axios调用后端RESTful API。

后端 (Backend)： Spring Boot应用程序，提供RESTful API接口，处理业务逻辑、数据持久化和安全性。

数据库 (Database)： MySQL存储持久化数据，Redis作为缓存和临时存储。

通信： 前端与后端通过HTTP/HTTPS协议进行数据交互，数据格式为JSON。

四、 功能模块设计
系统主要分为以下几个核心模块：

系统管理模块

用户管理： 对系统所有用户（教师、管理员等）进行增删改查、重置密码。

角色管理： 定义系统角色（如：超级管理员、科研秘书、教师、学院领导）。

权限管理： 基于RBAC模型，为角色分配菜单和API接口权限。

菜单管理： 动态配置系统导航菜单。

操作日志： 记录用户的关键操作，便于审计。

科研项目管理模块

项目申报： 教师在线填写项目申报书，上传附件，提交审核。

项目审批： 科研秘书、学院领导等多级在线审批流程。

项目立项： 审批通过后，项目正式立项，分配唯一项目编号。

项目中检： 记录项目中期检查情况和报告。

项目结题： 在线提交结题材料，进行结题验收。

项目查询： 按状态、类别、时间等多维度查询项目信息。

科研成果管理模块

论文管理： 教师登记发表的学术论文（标题、期刊、级别、作者等），支持附件上传。

著作管理： 登记出版的学术专著、教材。

专利管理： 登记申请的专利（发明专利、实用新型等）。

获奖管理： 登记获得的科研奖励。

成果审核： 科研秘书对教师登记的成果进行审核确认。

成果统计： 按个人、学院、时间范围进行成果统计。

科研经费管理模块

经费到账登记： 财务或科研秘书登记项目的到账经费。

经费使用登记： 项目负责人登记经费的预算和支出明细。

经费预算控制： 系统根据预算对支出进行预警。

经费报表： 生成经费使用情况的统计报表。

学术活动管理模块

学术会议登记： 教师登记参加的国内外学术会议。

学术报告管理： 登记校内举办的学术讲座或报告。

数据统计与可视化模块

个人工作台： 为每个角色展示相关的待办、通知、统计图表。

综合统计： 利用ECharts图表，可视化展示全院/全校的科研项目、经费、成果的统计与分析。

六、 核心实现步骤
环境搭建与项目初始化

使用 Spring Initializr 创建Spring Boot 3项目，勾选Web, Security, MyBatis, MySQL, Redis等依赖。

使用 npm create vue@latest 创建Vue 3 + TypeScript项目。

配置前后端项目的开发环境（IDE、数据库、Redis等）。

后端核心功能实现

实体类与数据层： 使用MyBatis-Plus，为每个核心表创建Entity和Mapper接口。

统一响应封装： 创建Result类，统一所有API接口的返回格式。

全局异常处理： 使用@ControllerAdvice创建全局异常处理器，返回友好的错误信息。

安全框架整合：

实现基于Spring Security + JWT的登录认证。

创建JwtAuthenticationTokenFilter，在请求前验证JWT Token。

实现UserDetailsService从数据库加载用户和权限信息。

使用@PreAuthorize注解和自定义权限校验器实现方法级别的授权。

业务逻辑开发： 按模块逐个开发Controller, Service, ServiceImpl。

API文档： 集成Knife4j，自动生成在线API文档。

前端核心功能实现

项目配置： 配置Vite，设置API请求基地址、路径别名等。

路由与布局： 使用Vue Router配置路由，设计主布局（包含头部、侧边栏、内容区）。

状态管理： 使用Pinia管理用户登录状态、权限信息等全局数据。

HTTP请求封装： 使用Axios创建请求实例，并添加请求/响应拦截器，用于自动携带Token和处理通用错误。

权限控制：

路由守卫： 在进入页面前校验用户权限。

按钮级别权限： 根据Pinia中存储的用户权限信息，自定义v-permission指令来控制按钮的显示/隐藏。

页面组件开发： 使用Element Plus组件，按功能模块开发各个页面。

联调与测试

前后端联动，测试所有功能接口。

进行性能、安全性和兼容性测试。

部署上线

前端： 执行npm run build打包，将dist目录部署到Nginx。

后端： 打包成JAR文件，使用java -jar命令运行，或使用Docker容器化部署。

配置Nginx反向代理，将API请求转发到后端服务。
